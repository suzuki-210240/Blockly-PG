基本的なデータ構造について

データ構造とは、データを効率的に管理、格納、整理するための方法や形式のことです。適切なデータ構造を選択することで、プログラムの性能と可読性を大幅に向上させることができます。 本節ではで、基本的なデータ構造(変数、配置,リスト、スタック、キュー、スタック)について学びます。

１，変数とは？

変数は、データを入れる箱のようなもので、変数名を付けて区別します。

変数はプログラムで使用するデータを一時的に格納するために用いられ、 データを使用する際には、変数名を指定して、処理をします。

２，変数の代入について

変数にデータを格納することを 「代入」 といい、 数値や文字、計算結果、他の変数の値を、代入することができます。 代入を示す書式は、 変数名 = 値;　または　代入先変数名 = 式;です。 


 
【例】
x = 10; // xに10を代入  
(変数名 = 値;)            
y = x + 5; // xに5を加えた値をyに代入
(代入先変数名 = 式;)
z = y; // yの値をzに代入 
(代入先変数名 = 代入元変数名;)

代入先の変数に元々格納されていたデータは上書きされ、 新しいデータが格納されます。 また、変数どうしの代入の場合は、代入元の変数の値が 代入先の変数にコピーされます。（図１）※代入元の値は変化しません ※値が変化しない数値や文字列を定数といいます。

3、 配列の基本

配列は、同じデータ型の複数の変数を連続して配置するデータ構造です。 変数というデータを入れる箱が連なったものだと考えるとわかりやすいでしょう。 各変数を要素を結び、要素が横方向に並ぶ配列を「1次元配列」といいます。 また、縦と横に並ぶ配列を「2次元配列」といいます。(図２) 要素の指定は、配列につけた配列名と要素の位置を示す添字（インデックス）を指定します。 添字が０からか、１から始まるかは問題によって異なるため注意しましょう。

配列の宣言

配列は、以下のように宣言します: let(int) arr = [1, 2, 3];

配列のアクセス

配列の要素には、インデックスを使用してアクセスします: arr[0]

配列の操作

配列の要素を変更するには、以下のようにします: arr[0] = 10; 

 
    【要素の指定の書式】
    1次元配列の場合：配列名[インデックス]
    2次元配列の場合：配列名[行のインデックス][列のインデックス]    
    # Pythonでの配列（リスト）の例
    fruits = ["リンゴ", "バナナ", "オレンジ"]
    print(fruits[0])  # インデックスで要素にアクセス

４，リストについて

リストは、データとポインタを１つの要素として持ち、ポインタで要素間をつなげたデータ構造で、 配列と同様に複数の要素を格納することができます。

リストは、Pythonなどのプログラミング言語で使用されるデータ構造で、要素の追加や削除が容易です。

５，リストの構造

リストはデータを連結したデータ構造で、リストの最小単位となる要素は「データ部」と「ポインタ部」から構成されます。
データ部には、リストに格納するデータが格納され、ポインタ部には次の要素のアドレス(住所)が格納されます。 このポインタをたどることで、個々の要素にアクセスすることができます。
リストの種類には、ポインタの向きによって、 「単方向リスト」、 「双方向リスト」、 「循環リスト」などがあります。（図３）


　

リストの種類

種類 内容
単方向リスト 次の要素を示すポインタのみを持つリスト。 先頭から末尾の方向へデータをたどることができる
双方向リスト 次の要素と前の要素を示す2つのポインタ持つリスト。 先頭から末尾、末尾から先頭の方向へデータをたどることができる
環状リスト 末尾の要素のポインタが先頭の要素を示すリスト。 要素が環状に連結される

※薄い青はデータ部、濃い青はポインタを指します

５，リストの操作

リストで要素の追加や削除を行う場合は、配列と比べて簡単に操作を行うことができます。 配列では、要素の追加や削除を行う場合、要素の移動が発生するため、処理が複雑になり時間がかかりますが、 一方リストは、ポインタの参照先変更するだけで良いので、短時間での要素の追加や削除が可能です。 例として、図4に示す英単語をアルファベット順に格納した配列とリストに対して要素の追加と削除をします。

６、 キューについて

キューは、先に格納したデータから順に取り出す、先人先出 型（FIFO：First In First Out）のデータ構造です（図2.7）。 キューヘデータを格納することを enqueue（エンキュー）といい、 キューからデータを取り出すことを dequeue（デキュー）と呼びます。

図７：キュー



７、 スタックについて

スタックはキューとは逆に、後に格納したデータから順に取り出す， 後入先出型（LIFO：Last In First Out）のデータ構造です。 スタックにデータを格納することをpush（ブッシュ）。 スタックからデータを取り出すことをpop（ポップ）と呼びます。 スタックは、サブルーチンや再帰処理における、 戻り先アドレスや呼出し時点のデータの一時的な格納に利用されます。
